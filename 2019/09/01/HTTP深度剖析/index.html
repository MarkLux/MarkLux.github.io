入门

  1. http协议是建立在TCP/IP协议基础上的一个应用层协议

    所有有关数据传输的事情都与TCP/IP有关

  2. 所有的www文件都是基于http协议发送的

  3. http目前常用的为1.1版本（长连接）

  4. [举例]打开一个网页时要发送多少请求？

    这里要注意：文本，图片，音频，css，js等都是分开请求加载的，因此打开一个页面将会发送多个http请求

HTTP报文

基本结构

  1. 请求行/状态行
  2. 消息头
  3. 一个空行
  4. 实体内容

请求报文说明

  • 请求行：

    请求行说明请求的方法，请求的资源以及协议版本

  • 常用消息头：

    1. Accept: 表示客户端所能接收的数据类型

    2. Referer: 表明请求的来源，也就是引导用户到当前页面的那个页面的地址（如果有的话）

    3. Accept-Language： 表示能读取的语言类型

    4. Accept-Encoding: 表示能接收的压缩格式（数据是通过压缩包来传输的）

    5. User-Agent: 说明了浏览器的版本，内核以及操作系统

    6. Host: 主机地址

    7. Connection: 表示连接类型（有长连接和短连接两种）

响应报文说明

  • 状态行：
    包含了HTTP协议的版本和响应的状态码

  • 常见状态码及实例说明：

    [分类]：
    2XX状态码表示请求成功，3XX表示重定向，4XX表示客户端错误，5XX表示服务器端错误

    1. 200 OK 表示请求成功

    2. 302 重定向

      重定向(Redirect)就是通过各种方法将各种网络请求重新定个方向转到其它位置，一般的流程是，客户端向服务器端发出一个请求，服务器端返回一个重定向要求(Laction)，然后浏览器向重定向到的那个页面发出请求，实例可以参考下面PHP与HTTP的header()函数。

    3. 304 Not modified

      服务器端表示请求的资源并没有更新过，要求浏览器直接去读取缓存，实例见下面响应实例

    4. 404 找不到页面

  • 常见消息头：

    1. Date: 表示请求的页面的时间（如果没有到更新时间浏览器将会自动从缓存当中取出页面）

    2. Server: 服务器主机情况

    3. Content-Length: 报文内容的大小（字节，注意是内容，不包括报文的头消息等）

    4. Content-Type： 返回内容的类型

    5. Location: 重定向页面

    6. Content-Encoding: 页面编码支持的压缩算法

    7. Last-Modified: 表示浏览器所请求的资源的最新更新时间

    8. Refresh:告诉浏览器间隔多少时间就重定向到一个页面去（见实例）

    9. Cache-control： 操作缓存机制

    10. Expires: 操作缓存机制

    11. Pragma: 操作缓存机制

    12. Content

一、304与缓存

假设我们现在访问页面a.html:

在浏览器中输入对应的地址进行访问，可以看到发送了两次请求，两次的回应都是200 OK，其中第一次取回了html文档内容，
第二次取回了test.png这个文件。取回这张图片之后，浏览器端将会给它设定一个Max-Age参数，只要你不再向服务器端发
出请求(如刷新)，那么浏览器在打开这个页面时就不会再向服务器发出任何请求，而是直接读取本地缓存。

现在我们刷新界面，可以看到又发送了两次请求，两次的回应一个是200 OK，一个是304 Not modified。第一次仍然取回
html文档内容，浏览器解析到图片时，就查看缓存，发现该图片已经在缓存中了，就向服务器端发出请求，问我要不要更新这
个图片，服务器端接收请求后发现该文件并没有发生改动就返回304让浏览器直接读取缓存了。
